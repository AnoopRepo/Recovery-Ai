# Recovery Ai 🌿

**Recovery Ai** is a cutting-edge, AI-driven platform designed to help professionals and students identify, analyze, and recover from digital burnout. Using a personalized assessment powered by Google's Gemini LLM, it provides structured recovery plans tailored to individual needs.

![Recovery Ai Screenshot](https://images.unsplash.com/photo-1506126613408-eca07ce68773?ixlib=rb-4.0.3&auto=format&fit=crop&w=1200&q=80)

## ✨ Features

- 🧠 **AI-Powered Assessment**: A comprehensive quiz that analyzes your digital habits and emotional state.
- 📊 **Personalized Recovery Score**: Get a score out of 100 indicating your current burnout severity.
- 📋 **Structured Recovery Plans**: Receive 3-5 actionable steps generated by AI to help you regain balance.
- 🔐 **Secure Authentication**: JWT-based login and signup to keep your health data private.
- 📱 **Responsive Design**: A beautiful, calming UI built with a mobile-first approach.
- 🕰️ **History Tracking**: View your past assessments and track your recovery journey over time.

## 🛠️ Tech Stack

### Frontend
- **React (Vite)**: For a lightning-fast user experience.
- **Tailwind CSS**: Modern, utility-first styling.
- **Framer Motion**: Smooth, premium animations.
- **Lucide React**: Minimalist and beautiful iconography.
- **React Markdown**: Clean rendering of AI-generated content.

### Backend
- **Spring Boot**: Robust and scalable Java framework.
- **MongoDB**: NoSQL database for flexible data storage.
- **Google Gemini API**: State-of-the-art LLM for personalized analysis.
- **Spring Security + JWT**: Enterprise-grade security for user authentication.

## 🚀 Getting Started

### Prerequisites
- Java 17 or higher
- Node.js 18 or higher
- MongoDB Atlas account (or local MongoDB)
- Google Gemini API Key

### Backend Setup
1. Navigate to the `backend` directory.
2. Create/Update `src/main/resources/application.properties`:
   ```properties
   spring.data.mongodb.uri=your_mongodb_uri
   jwt.secret=your_jwt_secret
   llm.api.key=your_gemini_api_key
   ```
3. Run the backend:
   ```bash
   ./mvnw spring-boot:run
   ```

### Frontend Setup
1. Navigate to the `frontend` directory.
2. Install dependencies:
   ```bash
   npm install
   ```
3. Create a `.env` file:
   ```env
   VITE_API_URL=http://localhost:8080
   ```
4. Run the development server:
   ```bash
   npm run dev
   ```

## 🌐 Deployment

This project is prepared for deployment on modern platforms:
- **Backend**: Render, Railway, or Heroku.
- **Frontend**: Vercel or Netlify.

*Ensure you set the `ALLOWED_ORIGINS` and `VITE_API_URL` environment variables in your production environment.*
